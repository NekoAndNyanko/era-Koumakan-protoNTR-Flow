　PRINT_STATE.erb、OPTION.erb、のそれぞれにIRUINAME_SMELLに関する記述を追加。

・検索の際はSMELLやIRUIで検索すると楽でしょう。

・１日〜１０日着用までを１日〜３０日の間で順番に表示。
　つまり、１日、１１日、２１日、になると自動的に１日着用済みとなる
　(DAY:2が1,11,21のときは１日、2,12,22の時は２日、という扱いにしているということ)
　なので、同じ服を着用していなくても三日着用済みとかなる。
　ぶっちゃけそれって、ただ風呂入ってないだけじゃね？
　とかいう質問は受け付けない。だって、訪問者とめーりんが風呂入ってたし(泣)
　きっと洗わないで取っておいて、履いているんです。そう思って下さい。

・能力欄にはパンティ以外も出るけど、エロしてるときに脱がす行為への反映は、現在パンティのみです。

・改造、改変、引用、修正、吸収合併、改悪、改善、廃棄、他パッチと合体やまとめる、何でも好きにしてくれ。

☆注意点＆余談☆

・CLOTHES.erbの@IRUINAME_SMELL(旧@PANTSNAME_SMELL)の式を、全部、ELSEIFでやっちゃったから、被っている部分が表示されない、
または二重に表示される可能性あり……という状況だったが、二重に表記されるであろうNTR部分はカットしたお！

もともと、NTR+精液残留or子宮内精液、というのも作ろうと思ってたので、
式を適当に書くと

	IF DAY:2
　		IF TALENT:NTR && CFLAG:17
		ELSE
		ENDIF
	ELSE
	ENDIF

みたいな感じでやりたかったんだが、
NTRの対象は誰をだせばいいのかとか、
子宮内精液は誰の子種？　つーか、もしかしてこれ
ステータス？か何かで表示するときに、CALLして計算した結果だしていて、
この数値出しました→表示なのか？　いや、違うかもしれん……


考えてたら面倒になったので、とりあえず精液残留で様子見するしかないか……
って感じで投げ捨てた。
ぶっちゃけ、これ以降修正とかやる気でねぇので、
不具合出たら勝手に修正しちゃっていいですぅ


という感じで捻り回してたら、やる気が千切れたお！
